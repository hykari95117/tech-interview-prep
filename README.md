# Backend / Enterprise System Interview Question List

---
## 0. Overview
- 컴퓨터 사이언스 or 주니어 수준에 어울리는 기술 질문보다는 경력 즉, 어떤 경험을 가졌는지를 더 중요하게 보고 집중적으로 질문 예상
- 물론 CS, 기술질문 등도 대비는 해야한다.
- 사람과의 관계(OO가 힘들어서 퇴사), 인간관계에 대한 얘기는 아예 하지 않는다는 생각으로 임하자

---

## 1. Technical – General
- 본인의 주요 역할과 담당 범위를 설명해 주세요
- 가장 자신 있는 기술 스택은 무엇인가요?
- 설계에 직접 참여한 경험이 있나요?
- 레거시 코드를 개선한 경험이 있다면 설명해 주세요
- 운영 환경에서 개발 시 중요하게 생각하는 점은 무엇인가요?

---

## 2. Backend / Spring
- Spring Framework 또는 Spring Boot 사용 경험을 설명해 주세요
- Spring Data JPA 사용 시 장단점은 무엇이라고 생각하나요?
- JPA N+1문제 발생 원인부터 해결 방법
- JPA 사용 중 성능 또는 동시성 이슈를 겪은 적이 있나요?
- 트랜잭션 관리는 어떻게 하고 있나요?
- 대용량 데이터 처리 시 고려하는 사항은 무엇인가요?
- 성능 튜닝 경험이 있다면 어떤 방식으로 접근했나요?

---

## 3. Database / SQL
- RDBMS 기반 SQL 작성 경험을 설명해 주세요
- 인덱스 설계 또는 쿼리 튜닝 경험이 있나요?
- ORM과 Native SQL을 선택하는 기준은 무엇인가요?
- 데이터 정합성을 유지하기 위해 고려한 사항은 무엇인가요?

---

## 4. Frontend Collaboration
- 프론트엔드와 협업한 경험이 있나요?
- 프론트엔드 성능 이슈를 함께 해결한 경험이 있나요?
- 사용자 경험 개선을 위해 고려한 사항은 무엇인가요?

---

## 5. DevOps / Operation
- 형상관리 도구 사용 경험을 설명해 주세요
- 빌드 및 배포 프로세스에 참여한 경험이 있나요?
- 운영 환경에서 장애가 발생했을 때 대응 방식은?
- 배포 리스크를 줄이기 위해 어떤 노력을 했나요?
- 운영 중 반복 업무를 개선한 경험이 있나요?

---

## 6. Security / Stability
- 웹 보안과 관련된 대응 경험이 있나요?
- 사용자 입력값 검증은 어떻게 처리하나요?
- 서비스 안정성을 위해 고려한 설계 요소는 무엇인가요?
- 장애 재발 방지를 위해 어떤 조치를 했나요?

---

## 7. Enterprise / Business Domain
- 업무 프로세스를 시스템으로 구현한 경험이 있나요?
- 복잡한 비즈니스 로직을 설계할 때 고려하는 점은 무엇인가요?
- 권한 또는 역할 기반 설계 경험이 있나요?
- 외부 시스템과 연동한 경험이 있다면 설명해 주세요
- 업무 시스템 개발에서 가장 중요하다고 생각하는 요소는 무엇인가요?

---

## 8. AI / Productivity Tools
- 개발 생산성을 높이기 위해 사용하는 도구가 있나요?
- 코드 자동 생성 도구를 사용할 때 주의하는 점은 무엇인가요?
- 도구 사용 결과를 어떻게 검증하나요?

---

## 9. Non-Technical – Work Style
- 본인의 강점과 약점은 무엇인가요?
- 협업 시 중요하게 생각하는 가치는 무엇인가요?
- 의견 충돌이 발생했을 때 어떻게 해결하나요?
- 반복적이거나 운영 중심 업무에 대한 생각은?
- 스트레스는 어떻게 관리하나요?

---

## 10. Motivation / Career
- 이직 또는 지원 동기는 무엇인가요?
- 조직을 선택할 때 중요하게 보는 기준은?
- 장기적인 커리어 목표는 무엇인가요?
- 3~5년 후 본인의 모습을 어떻게 그리고 있나요?
- 입사 후 (직무적인) 목표
- 많은 회사 중에 왜 우리 회사인가?
- 다른 회사는 어디 지원했는지
- 입사 후 직무와 환경이 생각한 것과 다르다면?

---

## 11. Closing
- 마지막으로 하고 싶은 말이 있나요?
- 면접관에게 질문하고 싶은 점이 있나요?
